# nest-boost

**Laravel Boost, for NestJS.** An MCP server + AI guidelines + agent skills that
teach your coding agent (Claude Code, Cursor, Codex, Gemini) how *your* NestJS app
is actually wired — its real modules, routes, and DI graph — and how to write
idiomatic Nest code. Built for [Bun](https://bun.sh) as runtime and package manager.

## Install

```bash
bun add -d nest-boost
bunx nest-boost install
```

`install` detects your NestJS ecosystem packages, then asks:
- **Architecture style** — Standard layered / CQRS / Hexagonal
- **Auth strategy** — Passport / Better Auth / none (defaulted from what you already depend on)
- **Agents** to configure

…and writes the MCP server config, composes AI guidelines, and installs skills. The
architecture and auth choices each tailor the guidelines and install a matching skill,
so the agent scaffolds code the way *your* project is structured. Pick **Better Auth**
and nest-boost also offers to pull the official community skill
(`npx skills add better-auth/skills`) and points the agent at
[`better-auth.com/llms.txt`](https://better-auth.com/llms.txt).

Non-interactive: `bunx nest-boost install --agents claude --arch cqrs --auth better-auth --fetch-auth-skill`.

Keep resources fresh after dependency changes:

```bash
bunx nest-boost update
```

## What you get

### MCP server (`nest-boost mcp`)
Boots your app in **preview mode** (`NestFactory.create(AppModule, { preview: true })`)
— it reads the real DI container **without** running lifecycle hooks or opening
DB/network connections. Tools:

| Tool | Purpose |
| --- | --- |
| `application_info` | Bun/Node/Nest versions, detected packages, module/route/provider counts |
| `list_routes` | Every HTTP route: method, path, controller, handler, module, guards/interceptors/pipes |
| `module_graph` | Modules with their controllers, providers (scope + exported), imports, exports |
| `nest_cli` | List and run whitelisted `nest` CLI commands (generate, build, info) |

### AI guidelines
Composed from the packages you actually use (core Nest conventions + TypeORM,
Prisma, GraphQL, config, validation, auth, testing, …) and written to `CLAUDE.md` /
`AGENTS.md` / Cursor rules between managed markers — your own content is preserved.

### Agent skills
On-demand [Agent Skills](https://agentskills.io): `nestjs-development`,
`nestjs-docs` (authoritative lookups against the official docs on GitHub),
`testing-jest`, package-gated skills like `typeorm-development`, an architecture skill
(`architecture-standard` / `-cqrs` / `-hexagonal`), an auth skill
(`auth-passport` / `auth-better-auth`) matching your choices, and `skill-builder`
(below).

### Growing the knowledge base

nest-boost can't ship a skill for every library — so it ships one that **writes new
skills on demand**. Point the `skill-builder` skill at any dependency ("make a skill for
drizzle-orm") and the agent researches it (the library's `llms.txt`, official docs,
Context7, `npx skills add …`, its NestJS integration guide), writes
`.nest-boost/skills/<name>/SKILL.md`, and runs `nest-boost update` to propagate it to
every configured agent.

`.nest-boost/skills/` is your **committed** knowledge base (source of truth); the copies
under `.claude/skills/` etc. are regenerated by `update`. A local skill overrides a
packaged one of the same name.

## Configuration

`nest-boost.json` records the entry module and configured agents:

```json
{ "entryModule": "src/app.module.ts", "moduleExport": "AppModule", "agents": ["claude"] }
```
