# Guidelines & skills

nest-boost installs two kinds of knowledge into your agent: **guidelines** (loaded up front)
and **skills** (loaded on demand).

## Guidelines

Concise Markdown, composed from what your project uses and your install choices:

- core Nest conventions + a version note (e.g. Nest 11)
- your **architecture** section and **auth** section
- your **test-layout** section
- a **monorepo** section (when `nest-cli.json` declares a workspace)
- one section per detected ecosystem package (TypeORM, Prisma, GraphQL, Mongoose, config,
  validation, Swagger, queues, testing, тАж)

They're written into each agent's guideline file тАФ `CLAUDE.md`, `AGENTS.md`, a Cursor rule тАФ
**between managed markers** (`<!-- nest-boost:begin -->` тАж `end`), so your own content around
the block is preserved and the block is regenerated by `update`.

## Skills

[Agent Skills](https://agentskills.io) are folders with a `SKILL.md` (name + description
frontmatter) that agents load only when relevant. nest-boost installs:

- **Baseline** (always): `nestjs-development`, `nestjs-docs` (authoritative lookups against the
  official NestJS docs on GitHub), `testing-jest`, `suites-testing` (when Suites is present),
  `skill-builder`, `using-evaluate`.
- **Package-gated**: ORMs/ODMs (`typeorm-`, `prisma-`, `sequelize-`, `mikro-orm-`,
  `mongoose-development`), plus `graphql-development`, `orpc-development`, тАж тАФ installed only
  when the package is detected.
- **Architecture**: `architecture-standard` / `-cqrs` / `-hexagonal`.
- **Auth**: `auth-passport` / `auth-better-auth`.

Skills are copied into each agent's skills directory (`.claude/skills/тАж`); those copies are
regenerated by `update`.

## The knowledge base тАФ `.nest-boost/skills/`

Your own custom skills live in `.nest-boost/skills/<name>/SKILL.md`. This directory is the
**source of truth** and should be **committed**. `install`/`update` copy it into every
configured agent. A local skill overrides a packaged one of the same name.

## Growing it тАФ the `skill-builder` skill

nest-boost can't ship a skill for every library, so it ships one that **writes new skills on
demand**. Point the `skill-builder` skill at any dependency ("make a skill for drizzle-orm")
and the agent researches it (its `llms.txt`, official docs, Context7, an existing community
skill, its NestJS integration guide), writes `.nest-boost/skills/<name>/SKILL.md`, and runs
`nest-boost update` to propagate it.

## Skills bundled by a package

A library can ship its own skill and have nest-boost install it when the library is a
dependency (Laravel Boost's third-party package skills). Declare it in the library's
`package.json`:

```json
{ "nestBoost": { "skills": ["skill"] } }
```

Each listed directory holds `<skill-name>/SKILL.md`. Without the field, nest-boost also scans
the conventional roots `nest-boost/skills/`, `skills/`, and `skill/`.

## Priority

When the same skill name comes from multiple sources, the highest priority wins:

**packaged** тЖТ **package-bundled** тЖТ **local (`.nest-boost/skills`)**
