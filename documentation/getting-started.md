# Getting started

## Requirements

- **Node.js ≥ 18.18**
- A **NestJS** project (≥ 9) whose `tsconfig.json` has `emitDecoratorMetadata: true` (the Nest
  default) — the MCP server loads your app's TypeScript through [`tsx`](https://tsx.is) under
  Node, which needs that flag for NestJS dependency injection to resolve.
- Optional, only for the database tools: Node ≥ 22 (built-in `node:sqlite`) or `better-sqlite3`,
  and your app's DB driver (`pg`, `mysql2`, `mongodb`).

## Install

```bash
npm install -D @mgvdev/nest-boost
npx @mgvdev/nest-boost install
```

`install` inspects your project and asks a few questions:

- **Architecture style** — Standard layered · CQRS · Hexagonal
- **Auth strategy** — Passport · Better Auth · none (defaulted from your dependencies)
- **Test layout** — colocated · `__tests__/` · central `test/` (unit + feature)
- **Agents** — Claude Code · Cursor · Codex · Gemini CLI · generic

It then:

1. writes the MCP server configuration for each selected agent,
2. composes AI guidelines from the packages you use + your choices,
3. installs the matching skills.

See [Install options](./install-options.md) for what each choice does and the non-interactive
flags.

## First run

Open your agent (e.g. Claude Code) in the project. It picks up the generated `.mcp.json`
automatically and can now call the [MCP tools](./mcp-tools.md) — start by asking it to run
`application_info`.

Ask it to build something ("add a `cats` resource"): it follows the installed
[guidelines & skills](./guidelines-and-skills.md), and grounds itself with `module_graph` /
`list_routes` instead of guessing.

## Keeping it fresh

After changing dependencies, re-sync guidelines and skills:

```bash
npx @mgvdev/nest-boost update
```

## What gets written

| Path | What | Commit? |
| --- | --- | --- |
| `.mcp.json`, `.cursor/`, `.gemini/` | MCP server config | optional (regenerated) |
| `CLAUDE.md`, `AGENTS.md`, `.cursor/rules/` | guidelines (between markers) | optional |
| `.claude/skills/…` | installed skill copies | optional (regenerated) |
| `nest-boost.json` | nest-boost config | optional |
| `.nest-boost/skills/…` | your custom skills | **yes** — source of truth |

The generated agent files are regenerated by `update`; only `.nest-boost/skills/` is meant to
be committed. See [Configuration](./configuration.md).
