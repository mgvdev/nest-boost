## NestJS 12 Notes

- NestJS 12 ships as ESM-first. New projects default to ESM; the CLI still offers a
  CommonJS scaffold for gradual migration. Existing code should move toward ESM
  imports and review `tsconfig.json` module/moduleResolution settings.
- The CLI build pipeline uses **Rspack** instead of webpack. Audit custom webpack
  plugins and build scripts before upgrading.
- New ESM projects default to **Vitest** + **oxlint**; CJS schematics keep Jest.
  Existing Jest setups keep working, but fresh projects should adopt the new defaults.
- Route parameter decorators (`@Body()`, `@Query()`, `@Param()`, …) accept a
  `{ schema: ... }` option with any Standard Schema library (Zod, Valibot, ArkType).
  This is an alternative to `class-validator`, not a replacement.
- GraphQL projects use **GraphiQL** as the default playground.
- Keep `app.enableShutdownHooks()` for graceful termination in containers.
- Before generating code, check the actual Nest version with `application_info` and
  prefer the patterns that match the project's chosen module format.
