## NestJS 11 Notes

- Requires Node 20+. Express 5 is the default HTTP adapter — route wildcards use the
  named form (`*splat`) rather than a bare `*`.
- `@nestjs/config` and other packages ship as ESM-friendly; prefer ESM `import` syntax.
- Logger supports per-level configuration and JSON output; configure it at `NestFactory.create(App, { logger })`.
- `CacheModule` now builds on `cache-manager` v6 (Keyv) — check store configuration when upgrading.
- Prefer `app.enableShutdownHooks()` for graceful termination in containerised deploys.
